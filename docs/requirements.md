# VisitPro - 智能客户拜访管理系统 需求说明书

## 1. 项目背景
在销售管理过程中，业务员对客户的拜访记录往往零散且难以整理。VisitPro 旨在通过 AI 技术（Gemini & DeepSeek）简化这一过程，实现从语音录入到智能摘要、行动项提取及自动邮件撰写的全流程自动化，提升销售效率。

## 2. 目标用户
*   **销售代表**：记录拜访过程，生成跟进报告，管理客户信息。
*   **系统管理员**：维护团队成员，配置自定义字段，管理系统存储与 AI 模型。

## 3. 功能需求

### 3.1 仪表盘 (Dashboard)
*   **数据概览**：展示客户总数、本月拜访数、积极转化数及需关注项。
*   **活动分布**：以周为单位展示拜访活动趋势图。
*   **团队业绩**：展示团队成员的拜访量及成功率排名。
*   **近期追踪**：快速查看最近的拜访记录摘要及情绪反馈。

### 3.2 客户管理 (Client Manager)
*   **客户档案**：支持名称、公司、行业、状态（活跃/潜在/流失）及联系方式的管理。
*   **动态扩展**：支持管理员定义的自定义字段。
*   **模糊搜索**：支持按姓名或公司快速检索。

### 3.3 拜访管理 (Visit Manager)
*   **多维度记录**：支持文字录入和语音录入（集成科大讯飞转写）。
    *   **录入优化**：新增“参加人员情况”专用输入框，详细记录参会背景。
*   **交互增强**：
    *   **语音转写反馈**：转写过程中提供动态进度条和录音波纹动画。
*   **AI 智能分析**：
    *   **模型切换**：用户可选 Gemini 或 DeepSeek 模型进行分析。
    *   **自动摘要**：将凌乱的笔记整理为标准 CRM 摘要。
    *   **情绪识别**：分析客户在沟通中的情绪倾向（积极/中立/消极）。
    *   **行动项提取**：自动识别并列出后续待办事项。
*   **跟进邮件**：基于会议内容自动生成跟进邮件草案，并支持一键模拟发送。
*   **日历视图**：以日历形式直观展示历史和计划的拜访。

### 3.4 系统管理 (Admin Panel)
*   **团队管理**：管理成员角色（Admin/User）及部门信息。
*   **字段配置**：支持对客户、拜访、用户对象添加自定义属性。
*   **存储配置**：支持 LocalStorage（本地）与 MySQL（远程，需后端支持）切换。
*   **AI 配置**：配置 DeepSeek 等模型的 API Key。
*   **备份恢复**：支持全量数据的 JSON 导入与导出。

## 4. 非功能需求
*   **响应式设计**：适配 PC 及移动端浏览器。
*   **离线优先**：在本地存储模式下，无需持续联网即可使用核心功能。
*   **安全性**：API Key 仅存储于用户本地浏览器环境中。
